## KEY CODE PATTERNS — Quick Reference

### Gold Table DDL (DM-02, DM-03, DM-06, CM-02, CM-03)
```sql
CREATE TABLE IF NOT EXISTS ${catalog}.${schema}.dim_customer (
    customer_key STRING NOT NULL
        COMMENT 'Surrogate key. Business: Fact joins. Technical: MD5 of customer_id + effective_from.',
    customer_id STRING NOT NULL
        COMMENT 'Natural key. Business: Customer identifier. Technical: Source system ID.',
    customer_name STRING
        COMMENT 'Full name. Business: Reports. Technical: From silver_customers.',
    effective_from TIMESTAMP NOT NULL,
    effective_to TIMESTAMP,
    is_current BOOLEAN NOT NULL,
    CONSTRAINT pk_dim_customer PRIMARY KEY (customer_key) NOT ENFORCED
)
CLUSTER BY AUTO
COMMENT 'Customer dimension with SCD2 versioning.
Business: Customer segmentation and reporting.
Technical: MD5 surrogate key, incremental from Silver.';

-- FK on fact table (apply AFTER both tables exist)
ALTER TABLE ${catalog}.${schema}.fact_orders
ADD CONSTRAINT fk_orders_customer
FOREIGN KEY (customer_key) REFERENCES ${catalog}.${schema}.dim_customer(customer_key)
NOT ENFORCED;
```

### MERGE with Deduplication (DA-03)
```python
from delta.tables import DeltaTable
from pyspark.sql.functions import col, md5, concat_ws

# Step 1: Deduplicate Silver (CRITICAL — must match merge key)
silver_df = (
    spark.table(silver_table)
    .orderBy(col("processed_timestamp").desc())
    .dropDuplicates([business_key])
)

# Step 2: Generate surrogate key
silver_df = silver_df.withColumn(
    "surrogate_key",
    md5(concat_ws("||", col(business_key), col("effective_from")))
)

# Step 3: MERGE
DeltaTable.forName(spark, gold_table).alias("t").merge(
    silver_df.alias("s"),
    f"t.{business_key} = s.{business_key}"
).whenMatchedUpdateAll().whenNotMatchedInsertAll().execute()
```

### DLT with Expectations (DQ-01, SA-01)

```python
    name="silver_orders",
```

### TVF Pattern (TF-01, TF-03, TF-04, TF-05, TF-06)
```sql
CREATE OR REPLACE FUNCTION ${catalog}.${schema}.get_costs(
    workspace_name STRING,                        -- Required first
    start_date STRING DEFAULT '2024-01-01',       -- Optional after
    end_date STRING DEFAULT '2024-12-31',
    limit_rows STRING DEFAULT '100'
)
RETURNS TABLE (workspace STRING, total_cost DECIMAL(18,2), cost_rank INT)
COMMENT '
• PURPOSE: Cost summary by workspace.
• BEST FOR: Total spend | Workspace comparison
• NOT FOR: Daily drill-down (use get_daily_costs)
• RETURNS: PRE-AGGREGATED rows (one per workspace)
• PARAMS: workspace_name (required), start_date/end_date (YYYY-MM-DD)
• SYNTAX: SELECT * FROM get_costs(''engineering'')
• NOTE: DO NOT wrap in TABLE(). DO NOT add GROUP BY.
'
RETURN
WITH metrics AS (
    SELECT workspace_id, SUM(list_cost) as total_cost
    FROM ${catalog}.${schema}.fact_usage
    WHERE usage_date BETWEEN CAST(start_date AS DATE) AND CAST(end_date AS DATE)
    GROUP BY workspace_id  -- Single aggregation pass (TF-06)
),
ranked AS (
    SELECT w.workspace_name AS workspace, m.total_cost,
           ROW_NUMBER() OVER (ORDER BY m.total_cost DESC) as cost_rank
    FROM metrics m
    JOIN ${catalog}.${schema}.dim_workspace w
        ON m.workspace_id = w.workspace_id AND w.is_current = true
    WHERE w.workspace_name LIKE CONCAT('%', workspace_name, '%')
)
SELECT * FROM ranked WHERE cost_rank <= CAST(limit_rows AS INT);
```

### Metric View YAML (MV-01..08)
```yaml
version: "1.1"

    format: { type: currency, currency: USD }
```

## COMMON ERRORS — Quick Fix

| Error | Cause | Fix |
|-------|-------|-----|
| `Column 'X' not found` | Hardcoded column name | Extract from Gold YAML |
| `DELTA_MULTIPLE_SOURCE_ROW_MATCHING` | No dedup before MERGE | `.orderBy().dropDuplicates([key])` |
| `argparse error` in notebooks | Used argparse | `dbutils.widgets.get()` |
| `python_task` invalid serverless | Wrong task type | `notebook_task` in YAML |
| `Variable ${var.X} not found` | Missing variable | Define in `databricks.yml` variables |
| `Unrecognized field "name"` | Metric View has `name` | Remove — name = filename |
| `Permission denied` serving | Missing resources | Declare resources + `SystemAuthPolicy` |
| `NOT_A_SCALAR_FUNCTION` | TVF in `TABLE()` | Call directly: `SELECT * FROM tvf(...)` |
| `INVALID_LIMIT_LIKE_EXPRESSION` | Param in `LIMIT` | `ROW_NUMBER() WHERE rn <= CAST(p AS INT)` |
| `ModuleNotFoundError` import | Notebook header | Remove `# Databricks notebook source` |
| Cartesian product in TVF | Multiple reads of same table | Single aggregation pass |
| `Table does not have PK` | FK applied before PK | Apply PK constraints first, then FK |

## AGENT MODE PROTOCOL

In agent mode you can execute code to read detailed rule files. Follow this protocol:

### Step 1 — Identify Domains
Scan user code and request for trigger keywords from the Navigation Index.

### Step 2 — Load Domain Rules
```python
base = "/Workspace/Enterprise_Rules"
content = open(f"{base}/domain/{domain_name}.md").read()
```
For cross-domain code, load ALL relevant files. A Gold pipeline typically involves:
- `data_pipelines.md` (MERGE, dedup, YAML schema)
- `data_modeling.md` (PK/FK, star schema)
- `naming_tagging.md` (snake_case, comments, tags)
- `delta_lake.md` (CLUSTER BY AUTO, managed tables)

### Step 3 — Apply Rules
For each rule in loaded files:
1. Check if the rule applies to the code
2. If **violated**: report Rule ID, violating code, corrected code, business reason
3. If **compliant**: note as PASS (for audits only)

### Step 4 — Load Templates
When generating new code, load the appropriate template:
```python
template = open(f"/Workspace/Enterprise_Rules/examples/{template_name}").read()
```
Available templates: `gold_table_template.sql`, `dlt_pipeline_template.py`, `asset_bundle_template.yml`, `metric_view_template.yml`, `agent_template.py`

### Step 5 — Fallback
If `/Workspace/Enterprise_Rules/` files are not found, fall back to Top-40 rules and Key Code Patterns in this document. Inform user that detailed rules are unavailable.
